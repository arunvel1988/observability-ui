Logs in Observability

Logs are time-stamped records that provide detailed context about what is happening within a system.

They are especially useful for debugging, auditing, and understanding failures.

Logs in an E-commerce Website

  • Login attempts and failures
  • Cart updates and abandoned cart messages
  • Order placement success/failure logs
  • Error logs during payment gateway integration
  • Inventory service communication failures

Logs in Server Infrastructure

  • Authentication attempts to services
  • Unauthorized access logs (e.g., SSH login failures)
  • Application crashes or stack traces
  • Deployment events and CI/CD logs
  • Audit trails for config changes
  • Web server access logs (e.g., NGINX, Apache)
