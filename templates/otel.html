What is OpenTelemetry (OTel)?

OpenTelemetry (OTel) is an open-source observability framework for cloud-native software.

It provides APIs, SDKs, and tools to collect traces, metrics, and logs from applications and infrastructure.

OTel helps developers and operators understand system behavior, troubleshoot issues, and improve performance.

Why Use OpenTelemetry?

  • Standardizes how telemetry data is collected and transmitted
  • Supports multiple programming languages and platforms
  • Integrates with many backends like Jaeger, Prometheus, OpenSearch, and commercial APMs
  • Provides rich context to connect traces, metrics, and logs
  • Designed for microservices, distributed systems, and cloud-native apps

Components of OpenTelemetry

  • Instrumentation Libraries
    These are language-specific libraries that you add to your application code to generate telemetry data like traces, metrics, and logs automatically or manually.
  • SDKs
    Software Development Kits that configure how telemetry data is collected, sampled, and exported from your applications. They provide APIs and tools tailored for different languages.
  • Collector
    A standalone, vendor-agnostic agent or gateway that receives telemetry data from instrumented applications, processes it (e.g., batching, filtering, transforming), and exports it to backend systems.
  • Exporters
    Components that send telemetry data from the Collector or SDKs to various observability backends such as Jaeger, Prometheus, OpenSearch, or commercial APM platforms.
  • Context Propagation
    Mechanisms that pass tracing context across service boundaries, enabling distributed tracing across microservices and different components.

How OpenTelemetry Works

Apps instrumented with OTel SDKs produce telemetry data → sent to the OpenTelemetry Collector → Collector processes and exports data → Observability backends visualize it.

This allows you to monitor and troubleshoot your distributed apps with unified data.

Install OpenTelemetry Collector
