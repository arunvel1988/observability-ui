📊 Metrics in Observability

📌 What is a Metric?

A metric is a numeric value collected over time that helps you monitor and understand the health, performance, and availability of systems.

Each metric typically has:

  • A name (e.g., cpu_usage)
  • A timestamp
  • A value (e.g., 75.5%)
  • Optional labels/tags (e.g., server="web01")

📦 Examples of Useful Metrics

🛒 Application Metrics (E-commerce)

  • ✅ Orders per second
  • ⚡ API response time (latency)
  • 📦 Inventory levels
  • 💳 Payment success rate

🖥️ Server & Infrastructure Metrics

  • 🔥 CPU Usage – Tracks processor load (e.g., node_cpu_seconds_total)
  • 🧠 Memory Usage – Amount of memory used/free
  • 💽 Disk I/O – Read/write throughput on disk
  • 🌐 Network Traffic – Ingress and egress bytes
  • 📉 Service Uptime – Whether a service is running or not
  • 📊 Number of Active Connections – e.g., to a DB or API
