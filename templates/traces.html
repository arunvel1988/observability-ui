📈 Traces in Observability

Traces help visualize the journey of a request across different microservices, showing how long each service took and where bottlenecks might occur.

They are critical in understanding **latency**, **dependencies**, and **service health** in distributed systems.

🛒 Traces in an E-commerce Website

  • 📦 User adds product → request hits cart-service
  • 🧾 Checkout triggers order-service
  • 💳 Order calls payment-service
  • 🚚 Payment success invokes shipping-service
  • 🔁 All services traced in one flow with span/timeline

🖥️ Traces in Microservice Infrastructure

  • 📍 Entry-point services (e.g., API Gateway) initiate root span
  • 🔧 Each internal service adds its span with latency details
  • 🧩 Helps detect service-level performance degradation
  • ⏱️ Latency heatmaps in tracing tools (e.g., Jaeger, Tempo)
  • 📊 End-to-end request timeline for debugging and optimization
